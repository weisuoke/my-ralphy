## Ralph-loop 的基本玩法详解

Ralph-loop（也叫 Ralph Wiggum loop）是一种让 Claude（尤其是 Claude Code）实现自主迭代编码的技术。核心想法是：给 Claude 一个固定提示（prompt），让它反复执行任务、检查结果、自我改进，直到达到完成条件或迭代上限

### 为什么叫 “loop”？

传统用 Claude Code：你手动输入任务 → Claude 修改代码 → 你检查 → 再手动继续。
Ralph-loop：用插件或脚本自动化“循环”。Claude 尝试完成任务后想退出时，被“拦截”，自动重新输入相同提示。它会读取自己刚改的代码、测试失败日志、git 历史，继续改进。
结果：Claude 像“自己跟自己对话”一样迭代，直到输出指定信号（如 “DONE” 或 “<promise>COMPLETE</promise>”）。

**标准用法（Claude Code 插件版）：**

安装 Ralph Wiggum 插件（官方或社区版）。
输入命令：text/ralph-loop "你的任务描述，比如构建一个 Todo App" --completion-promise "DONE" --max-iterations 50
Claude 开始工作：
修改代码、运行测试。
如果测试失败，继续改。
成功或达到上限才停。

你可以设置好后直接睡觉（“睡前编码”），醒来检查结果。

适合场景：

有明确验证（如单元测试通过、lint 无错）。
TDD（测试驱动开发）或从零建项目。
不适合：需要人类审美/设计决策的任务。

风险控制：必须设 --max-iterations 防止无限循环烧 token。

### 你提到的点1：用 Claude Code 的 resume session 节省上下文

这是社区优化版 Ralph-loop 的高级玩法，主要由 @Chris_Towles 分享（他的博客和工具很详细）。

问题背景：

普通 Ralph-loop 每次迭代上下文会累积，容易爆 token 窗口（Claude 上下文有限）。
频繁重读整个代码库浪费 token，Claude 容易忘记之前的研究/决策。

解决方案：resume session + session markers

Claude Code 支持 resume session：会话历史保存在本地 JSONL 文件（~/.claude/projects/），可以用 --resume <session-id> 恢复。
Chris 的创新：用“标记”（marker）定位并恢复特定会话，实现上下文高效传承。

玩法步骤（基于他的 towles-tool 工具）：

生成 marker：
运行 tt ralph marker create，输出如 RALPH_MARKER_abc12345。

研究阶段：
先手动让 Claude 研究任务（读代码库、规划）。
提示 Claude：“研究完后输出这个 marker: RALPH_MARKER_abc12345”。
Claude 输出 marker 后退出，这一轮会话就“标记”好了。

添加任务：
tt ralph task add "实现某个功能" --findMarker RALPH_MARKER_abc12345

运行 loop：
tt ralph run
脚本自动：
查找带 marker 的会话 ID。
用 --resume 从研究会话“fork”新会话。
Claude 继承完整上下文，继续编码。


好处：

研究只做一次，上下文永久保存。
每个任务从相同研究会话分支，token 消耗大幅降低（避免重复读代码库）。
适合复杂项目：先研究架构，再多个任务并行 loop。
实测：Chris 用它一夜完成审 37 篇博客、改 WebSocket 等大任务。

安装他的工具（可选）：
text npm install -g @towles/tool
claude plugin marketplace add ChrisTowles/towles-tool
这比纯插件版更省 token、更可靠，尤其适合长时间无人值守。

### 结合 UI 包装让 AI “睡前编码”

https://x.com/antirival/status/2011969466112897119

核心想法：

不满足于命令行或 Claude Code 原生界面。
造一个图形化界面（类似网页 app），让你轻松排队任务，AI 后台自主运行 Ralph-loop。

从他的演示视频看：

UI 叫 “Claude”（紫色暗黑主题）。
左侧文件树、右侧代码编辑、底部终端。
你输入任务（如 “Build a todo app with React and Tailwind”）。
点击 “Merge” 或类似按钮，AI 开始自主：
创建文件、写代码、npm install、运行 dev server。
实时显示进度（终端输出、文件变化）。
全程像看 app “自己建自己”。

你设置好任务队列，关电脑睡觉，AI 继续跑 loop。

好处：

比命令行友好：可视化进度、易管理多任务。
真正“睡前编码”：排队任务 → 启动 → 睡觉 → 醒来有成品。
他公开问“Who wanna try it out?”，说明是分享版或即将开源。

类似玩法还有别人做的（如 Ralph Wiggum UI 的 Kanban 看板），本质都是在 Ralph-loop 上包一层 UI，让操作更丝滑。
总结：

基础 Ralph-loop 已经很强（让 AI 通宵干活）。
resume session + marker 是省 token 神器，适合大项目。
UI 包装是用户体验升级，真正实现“发任务 → 睡觉 → 收成果”。
